# Research Notes

Created: 2026-07-08

## Reference Repository Shape

The reference repository `Tikam02/DevOps-Guide` uses a broad folder-based learning format with top-level areas such as CI/CD, cloud providers, container orchestration, DevOps, infrastructure provisioning, Linux, networking, observability, interviews, and books.

This repository follows that navigable style while expanding the scope into:

- AWS, Azure, and GCP
- DevOps and DevSecOps
- AI, ML, MLOps, and Agentic AI
- Observability with Prometheus, Grafana, and OpenTelemetry
- Tooling such as GitHub, GitLab, Jenkins, SonarQube, Nexus, Artifactory, Jira, Confluence, and draw.io
- Learner contribution workflows

## Source Preference

Use sources in this order:

1. Official vendor documentation
2. Official open source project documentation
3. CNCF, OWASP, or Linux Foundation material
4. Trusted engineering blogs only when they provide practical labs

## Maintenance Rule

When contributors add URLs, they should update `logs/url-validation-log.md` for major resources and rely on the Link Check workflow for pull requests.
