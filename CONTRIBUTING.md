# Contributing Guide

Thank you for helping learners. This repo is meant to become a practical, community-maintained guide for Cloud, DevOps, DevSecOps, AI/ML, MLOps, Linux, and related tools.

## What You Can Contribute

- Beginner-friendly notes
- Advanced production patterns
- Commands and cheat sheets
- Troubleshooting logs
- Interview questions
- Architecture diagrams
- Project ideas
- Validated official documentation links
- Cloud cleanup and cost-saving steps

## Quality Rules

1. Prefer official documentation links.
2. Validate every URL before adding it.
3. Mention the validation date when adding many links.
4. Keep beginner notes simple and advanced notes precise.
5. Do not add pirated books, copyrighted dumps, leaked exam questions, or paid course mirrors.
6. For cloud labs, always include cleanup steps.
7. For security content, keep it defensive, educational, and legal.

## Link Format

Use this format:

```md
- [Tool or topic name](https://example.com/docs) - one-line reason why this is useful.
```

## Pull Request Checklist

- [ ] My contribution is in the correct folder.
- [ ] I validated all URLs I added.
- [ ] I added cleanup steps for labs that create cloud resources.
- [ ] I avoided copyrighted or exam-dump content.
- [ ] My writing is clear for learners.
